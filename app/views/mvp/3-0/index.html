Version notes

  • Iterations are based on feedback from Round 2 of UR, conducted on version 2 of MVP
  • Mockup of award notice letter (round 2 version used in UR) - the following updates were made to the letter for round 3 of UR:
    • separated changes from other things they can do online
    • add the word ‘only’ to be more explicit thats it for changes online
    • added on same line as bank details what to do for other types of change
  • JIRA ticket for prototype iterations
  • Added 'get help' content to bank detail question pages to see if users feel they would need it and if they can find the right contact details in the GOV.UK page
  • Added full GYSP declaration, exclamation mark and all, to the check answers page in order to see if users react negatively to it
  • Updated journey to skip the ID&V journey so we have more time in UR sessions to focus on our service. Will still be showing the register page to gague users understanding of signing in or creating a new account
  • Start page - Added mobile 'number' in start page registration info, so users didn't think they'd need a smartphone
  • Start page - Separated change and view content into separate lists to help make clear what the service can and can't do
  • Start page - Added sentence about how identity checks helps with fraud to help educate and encourage users to go through IDV process
  • Added 'by phone or post' to Contact Pension Service links across service to make clear we're not just sending users to another / different online form
  • Confirmation page - highlighted return link with inset styling and updated content so users will notice and understand this link will help them confirm their change has gone through
  • Your State Pension page - Added change link to every section with an explanation page that these changes will come in future. Want to see if that makes it clearer to users this is a service building up and if they may use it in the future.
